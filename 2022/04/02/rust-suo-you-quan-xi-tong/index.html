Rust 所有权系统

发表于 2022-04-02 分类于 Rust 阅读次数： Disqus：

Rust 中的所有权无需垃圾回收，即可保障内存安全。

1 什么是所有权

1.1 Rust 的所有权规则

  1. Rust 中的每一个值都有一个所有者（owner）。
  2. 值在任何时刻有且只有一个所有者。
  3. 当所有者（变量）离开作用域，这个值将被丢弃。

1.4 内存与分配

对于字符串字面值，可以在编译时明确其大小，并为其分配合适的内存，并且它被硬编码进可执行文件中，使得字符串字面值快速且高效。

对于String 类型，为了支持可变，可增长的文本片段，需要在堆上分配一块在编译时未知大小的内存来存放内容。这意味着：

  • 必须在运行时向内存分配器（memory allocator）请求内存。

  • 需要一个当我们处理完 String 时将内存返回给分配器的方法。

对于典型的 C/C++ 语言，需要我们手动去分配与释放内存。Rust 采取了一个不同的策略：内存在拥有它的变量离开作用域后被自动释放，如下所示：

当变量 s离开作用域，Rust 为我们调用一个特殊的函数。这个函数叫做 drop。我们可以在自定义类型中定义自己的 drop 函数，用于各项资源的释放，例如堆内存，套接字，文件描述符等。

可变引用最大的限制：在同一时间只能有一个对某一特定数据的可变引用。例如尝试创建两个 s 的可变引用的代码会失败：

之所以做这个限制，是为了避免 数据竞争，它可由以下行为造成：

  • 两个或更多指针同时访问同一数据。

  • 至少有一个指针被用来写入数据。

  • 没有同步数据访问的机制。

2.6 引用总结

引用的规则如下：

  • 在任意给定时间，要么 只能有一个可变引用，要么 只能有多个不可变引用。

  • 引用必须总是有效的。

3 Slice 类型

slice 允许引用一段连续的元素序列。且它是不可变引用，所以它没有所有权。当它指向的数据被销毁时，它也变成无效的了。

4 总结

所有权、借用和 slice 这些概念让 Rust 程序在编译时确保内存安全。Rust 语言提供了跟其他系统编程语言相同的方式来控制你使用的内存，但拥有数据所有者在离开作用域后自动清除其数据的功能意味着你无须额外编写和调试相关的控制代码。
